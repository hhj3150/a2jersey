# 아임웹 임베드 스니펫 사용법

이 디렉토리의 `.embed.html` 파일은 **아임웹(또는 임의의 외부 사이트)의 HTML 삽입 위젯**에 통째로 붙여넣을 수 있는 자급자족형 코드 스니펫입니다.

```
embed/
├── 180ml.embed.html   ← A2저지 헤이밀크 180ml
├── 500ml.embed.html   ← A2저지 플레인 요거트 500ml
├── 750ml.embed.html   ← A2저지 헤이밀크 750ml
└── README.md          ← 이 파일
```

상위 디렉토리의 `180ml.html`, `500ml.html`, `750ml.html`은 **디자이너 핸드오프용 풀스크린 standalone HTML**입니다. 두 트랙을 같이 유지합니다.

---

## 1. 아임웹에 삽입하기

1. 아임웹 관리자 → **쇼핑몰 → 상품 상세** → 상품 선택 → **HTML 삽입** 위젯 추가
2. 임베드 파일을 텍스트 에디터(VSCode, 메모장 등)로 열고 **전체 선택(Cmd/Ctrl+A) → 복사**
3. 위젯 편집창에 그대로 **붙여넣기** → 저장

> 위젯 편집창이 `<style>` / `<link>` 태그를 자동으로 떼어낸다면, **"순수 HTML 모드"** 또는 "코드 입력 모드"로 전환 후 붙여넣기.

스니펫은 단일 wrapper(`.a2jp-180` / `.a2jp-500` / `.a2jp-750`)로 격리되어 있어 **사이트의 기존 스타일과 충돌하지 않습니다**.

---

## 2. 이미지 경로 교체 (필수)

스니펫 안의 `<img src="images/...">`는 **로컬 상대경로**입니다. 아임웹에서는 작동하지 않으므로 절대 URL로 바꿔야 합니다.

### 단계

1. 다음 이미지를 **아임웹 미디어 라이브러리**에 업로드:
   - 180ml: `180ml_milk.png`, `180ml_full_label.png`
   - 500ml: `500ml_yogurt.png`, `500ml_full_label.png`
   - 750ml: `750ml_milk.png`, `750ml_full_label.png`
   - 공통 인증마크 4종: `cert_lowcarbon.png`, `cert_animalwelfare.png`, `cert_haccp.png`, `cert_happyfarm.png`
2. 아임웹이 발급한 절대 URL(`https://imweb.me/.../180ml_milk.png` 형태)로 src를 일괄 치환
3. 가장 빠른 방법: 텍스트 에디터의 **찾아바꾸기**로 `images/` → `https://your-cdn-url/...` 일괄 변경

이미지 원본은 상위 디렉토리의 `product-pages/images/`에 있습니다.

## 3. 인증번호·표시사항 변경

각 임베드 스니펫에는 다음 데이터가 텍스트로 들어 있어, 변경 시 해당 라인만 수정하면 됩니다.

| 항목 | 위치 |
|---|---|
| 인증번호 4종 | `<span class="id">` 라인 (Back callouts 01~04) |
| 표시사항 11항 | `<dl class="tbl">` 안 |
| 영양성분 8항 | `<table>` 안 |
| 보관·주의 | `<section class="warn">` 안 |

**현재 등록된 인증번호** (변경 시 3개 파일 모두 수정):
- 동물복지: `동물복지-10-30-6-1호`
- 저탄소: `제 2024-2-068호`
- HACCP: `제 2016-0-0718호`
- 가축행복: `가축행복-23-2-2호`

## 4. 좌표 미세조정 (Blueprint 지시선)

병 사진에 그어진 SVG 지시선의 끝점이 **실제 라벨 위 마크와 어긋날 경우**, 다음을 조정:

```html
<svg class="bp-lines" viewBox="0 0 100 100" preserveAspectRatio="none" aria-hidden="true">
  <path d="M28 9 L50 8"/>     ← 라인의 시작(28,9) → 끝(50,8) 좌표
  <circle cx="50" cy="8" r=".7"/>   ← 끝점 dot의 좌표
  ...
</svg>
```

- `viewBox`가 `0 0 100 100` 이므로 모든 좌표는 **% 단위**
- 좌측 callout(`.bp-call-l`) 라인은 `M28 ...`로 시작 (왼쪽 컬럼 끝)
- 우측 callout(`.bp-call-r`) 라인은 `M72 ...`로 시작 (오른쪽 컬럼 끝)
- 끝점 `L X Y`와 dot `cx="X" cy="Y"`는 같은 좌표로 맞춰야 합니다

callout 박스의 세로 위치는 `style="--y:43%"` 처럼 인라인 변수로 잡혀 있어, 라인 좌표와 함께 조정.

## 5. 컬러 액센트

| 제품 | 네임스페이스 | 액센트 | 분위기 |

## 6. 모바일 자동 적응

가로 880px 이하에서는:
- Blueprint의 SVG 지시선 자동 숨김
- 병 사진 → callout이 **세로 스택 리스트**로 전환
- 4컬럼 / 9컬럼 그리드 → 2컬럼 / 3컬럼으로 축소

별도 작업 없이 자동 동작합니다.
